# プロジェクト開発ルール

このプロジェクトでは、AI（Antigravity）とユーザーが円滑に共同開発（Vibecording）を行うためのルールを定めます。

## 1. 開発の基本姿勢
- **会話は日本語**: 基本的な意思決定や相談はすべて日本語で行う。
- **実装はTypeScript**: 型定義を徹底し、`any`禁止。堅牢なコードを目指す。
- **ドキュメント優先**: 実装よりも先に設計とドキュメント（`docs/`内）を整備する。

## 2. ディレクトリ・ファイル構造
- **画面別カプセル化**: 画面固有のコンポーネント、フック、スタイルは `src/screens/[ScreenName]/` 内のフォルダで完結させる。
- **1ファイル 200行目安**: 可読性を保つため、1ファイルは200行程度を目安とする。必要に応じて分割する。
- **docs/pages/**: 各画面の詳細仕様はここに集約する。

## 3. Git / GitHub 運用
- **コミットメッセージ**: `feat:`, `fix:`, `docs:`, `config:` などのプレフィックスを用いる（Conventional Commits）。
- **ユーザー情報**: `shikiuku@gmail.com` / `shikiuku` を使用する。
- **即時Push**: 重要な変更が完了した後は、リモートリポジトリへPushする。

## 4. プロジェクト特有のルール
- **iPad ランドスケープ固定**: iPadの横向き画面に特化したUI/UXを設計する。
- **Zustand管理**: グローバルなステートはZustandで一元管理し、Propsバケツリレーを避ける。
- **画像生成**: 必要に応じて `generate_image` ツールを使用するが、構造設計を優先する。
